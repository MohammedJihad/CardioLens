# Dataset

## Source
UCI **Cleveland Heart Disease** dataset (303 patient records, 13 clinical
features + target). This is the most widely used of the four UCI heart-disease
databases (Cleveland, Hungary, Switzerland, VA Long Beach).

- `raw/heart_cleveland.csv` — the raw 303-row CSV as obtained from a public
  mirror of the UCI Cleveland data.
- `processed/heart_processed.csv` — generated by `python -m src.data`
  (cleaned + corrected target). **Not** committed if you enable `.gitignore`;
  regenerate it from raw.

If `raw/heart_cleveland.csv` is missing, `src/data.py` raises a clear error.
Drop any CSV with the standard columns below at that path and re-run.

| column | meaning |
|---|---|
| age | age in years |
| sex | 1 = male, 0 = female |
| cp | chest pain type |
| trestbps | resting blood pressure (mmHg) |
| chol | serum cholesterol (mg/dl) |
| fbs | fasting blood sugar > 120 mg/dl (1/0) |
| restecg | resting ECG result |
| thalach | maximum heart rate achieved |
| exang | exercise-induced angina (1/0) |
| oldpeak | ST depression induced by exercise |
| slope | slope of the peak exercise ST segment |
| ca | number of major vessels colored (0–4) |
| thal | thalassemia result |
| target | raw label (see the inversion note below) |

## ⚠️ Important: the raw `target` label is inverted

The popular circulating version of this CSV ships with a `target` column whose
values are **flipped relative to its own data dictionary**. We did not take the
documentation on trust — we checked it against clinical reality:

| group | n | mean age | mean max-HR | mean oldpeak | exang rate | mean `ca` |
|---|---|---|---|---|---|---|
| `target = 0` | 138 | 56.6 | **139** | **1.59** | **0.55** | **1.17** |
| `target = 1` | 165 | 52.5 | 158 | 0.58 | 0.14 | 0.36 |

Lower maximum heart rate, higher ST depression (oldpeak), more exercise-induced
angina, and more diseased vessels **all** track `target = 0`. These are textbook
markers of heart disease, so **`target = 0` is the diseased group**.

`src/data.py` therefore builds the modelling target as:

```python
heart_disease = 1 - target   # 1 = disease present, 0 = absent
```

This makes the **positive class = disease**, so recall / sensitivity correctly
rewards catching sick patients. After cleaning (one duplicate row removed) the
dataset is 302 rows with a 45.7% disease rate.

### Per-dataset target mapping (no global flip)

The `1 - target` rule is **specific to this inverted Kaggle CSV** and must NOT be
applied to every source. Target derivation is declared per dataset in
`src/dataset_registry.py`:

| source | raw label | rule | derivation |
|---|---|---|---|
| `cleveland_kaggle` (active) | `target` ∈ {0,1}, inverted | `kaggle_inverted_binary` | `heart_disease = 1 - target` |
| original UCI (`cleveland_uci`, `hungarian_uci`, `va_uci`, `switzerland_uci`) | `num` ∈ {0..4} | `uci_num_severity` | `heart_disease = (num > 0)` |

Original UCI files encode disease *severity* 0–4 where `0 = no presence` and
`1–4 = presence`, so the correct binary is `(num > 0)` — applying `1 - target`
to those would mislabel every patient. The registry enforces the right rule per
source and refuses a binary rule on non-binary labels.
