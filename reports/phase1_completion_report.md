# Phase 1 — Completion Report: Evaluation Depth & Statistical Honesty

_Educational and research use only. Outputs are **model scores**, not diagnoses;
not a medical device; no clinical validation._

Phase 1 made the Cleveland model evaluation statistically honest and clinically
aware. No model pipeline change was required; every number below is generated by
a committed script and reproducible from a clean clone.

## Final Cleveland metrics (held-out test, n=61, threshold 0.50) — unchanged
Random Forest (calibrated): ROC-AUC **0.892**, PR-AUC 0.853, sensitivity 0.714,
specificity 0.848, F1 0.755, log-loss 0.425, Brier 0.137; confusion TP 20 ·
**FN 8** · TN 28 · FP 5.

## 1. Threshold analysis (leakage-free selection)
The 0.5 cut-off is only a default. The operating threshold is selected on
**training out-of-fold predictions only** (test labels never used), minimising a
cost that weights a false negative **5×** a false positive. The selected
threshold is **0.20**; evaluated once on the held-out test set it gives
sensitivity 1.00, specificity 0.61 (FN 0, FP 13). A separate held-out *sensitivity
table* is provided for transparency but was **not** used to choose the threshold.
→ `reports/threshold_selection_train_oof.csv`, `reports/threshold_metrics.csv`,
`reports/threshold_test_evaluation.csv`, `reports/threshold_report.md`,
`reports/figures/threshold_plot.png`.

| threshold | sensitivity | specificity | FN | FP |
|---|---|---|---|---|
| 0.20 (selected on train OOF) | 1.000 | 0.606 | 0 | 13 |
| 0.50 (default) | 0.714 | 0.848 | 8 | 5 |

## 3. Nested cross-validation (unbiased generalisation)
Outer 5-fold × inner 5-fold (model + hyperparameter selection over LogReg / RF /
SVM): ROC-AUC **0.913 ± 0.047**, recall 0.789 ± 0.141, balanced-acc 0.849 ±
0.061. Consistent with the held-out test. **The inner loop selected Logistic
Regression in 4 of 5 folds** — strong evidence that on this small data the simple,
interpretable model is the right choice. → `reports/nested_cv_results.csv`,
`reports/nested_cv_report.md`.

## 4. Calibration comparison (CV-based, cautious)
Compared on **training out-of-fold** (primary) and held-out test (secondary).
On training OOF, isotonic had the lowest Brier (0.119) but a **much worse
log-loss** (0.65 vs ~0.39) — a sign it over-fits the small calibration set with
over-confident probabilities. The methods are close and unstable, so **no method
is decisively best**; the deployed isotonic is defensible, but sigmoid or none
are equally reasonable. We do not overstate a winner.
→ `reports/calibration_report.md`, `reports/figures/calibration_comparison.png`.

## 5. Decision curve analysis (net benefit)
Acting on the model yields higher net benefit than "treat all" or "treat none"
across roughly **0.05–0.60** threshold probabilities — a teaching demonstration of
net-benefit reasoning, not clinical evidence. → `reports/decision_curve_report.md`,
`reports/figures/decision_curve.png`.

## 6. Error analysis
8 false negatives, 5 false positives. **Some misses are borderline near 0.5
(0.43, 0.43, 0.48) while others are lower-confidence (0.23–0.30).** A lower
threshold recovers the borderline ones, not necessarily the rest. Strictly
descriptive; the sample is far too small for clinical conclusions.
→ `reports/error_analysis.md`.

## 7. Learning curve
Validation ROC-AUC rises 0.892 → 0.911 then **plateaus** (last-step ≈ 0.000) with
a train–validation gap of ~0.09. More data would help only modestly; the binding
constraint is the dataset's ~300-row size — motivating the planned external-data
phase. → `reports/learning_curve_report.md`, `reports/figures/learning_curve.png`.

## Tests
`pytest` — all passing (existing + new: threshold schema, bootstrap schema,
calibration report present, no-empty-report guard).

## Remaining limitations
- Single small cohort; 61-row test set → wide CIs (now reported honestly).
- No external validation yet (planned, later phase).
- Cost weights (5:1) are illustrative, not clinically derived.
- The operating threshold (0.20) is selected on **training out-of-fold** data
  (no test leakage); the saved model still scores probabilities, and 0.20 is a
  documented screening recommendation, not hard-coded into the artifact.
